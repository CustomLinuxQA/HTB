# Docker API exposed (no authentication)

## Что это
Docker Engine API открыт по TCP (обычно `2375`) **без TLS и аутентификации**.

Фактически:
> Docker API = root-доступ к хосту

## Типичные признаки
- Открыт порт `2375`
- Ответ на:
```bash
curl http://<host>:2375/version
```

## Почему нет авторизации

Docker API:

- не имеет логина/пароля
- безопасность основана на:
    - TLS + client certs
    - или локальном unix-socket

Если TCP открыт без TLS → **любой может управлять Docker**.

## Что можно сделать

- Запуск контейнеров
- Чтение env/секретов
- Монтирование файловой системы хоста
- Получение root на хосте

## Базовая эксплуатация (идея)

1. Создать контейнер
2. Смонтировать `/` (Linux) или `C:\` (Windows)
3. Выполнить команды или прочитать файлы

## Пример проверки

`curl http://<host>:2375/containers/json`

## Когда встречается

- Dev / Test окружения
- Docker Desktop
- Неправильно настроенные серверы
- Внутренние сети

## Критичность

- CVSS обычно 9.0+
- Почти всегда → root
